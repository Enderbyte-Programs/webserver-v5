Single Use Keys: Software as a product; Users put a provided key and it is verified. Keys here are single use and may only be used once.
Standard Key List: Software as a product; Users put a provided product key and it is verified. In this mode, keys may be used by multiple people
Identifier based Keys: Software as a product;Keys are verified based on an identifier (like an email or username). This allows products to be verified across devices. Keys still will be stored in a list and need to be activated. Keys may only be used by one identifier.
Subscription: Software as a service: Users' identifiers are given expiry dates. Note that this setting has no actual key and is purely based on identifiers. Trial periods may also be configured.
Mixed: The product will use a mix of license types. You will be asked for the type with each key.



Activation controls whether keys can be populated with actuall accounts and people. If the product is not updated, all non-management actions will not work (no verification, etc)
